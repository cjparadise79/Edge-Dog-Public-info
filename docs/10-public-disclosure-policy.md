# Public Disclosure Policy

This repository is public-facing and documentation-only.

## Allowed Content

Allowed public content includes:

- product and project summaries
- architecture concepts
- public diagrams
- demo scripts
- high-level technical explanations
- partner-facing briefs
- roadmap descriptions
- safety model summaries
- public FAQ material

## Not Allowed

Do not publish:

- source code
- credentials
- API keys
- private IP addresses
- hostnames that expose private infrastructure
- SSH details
- internal deployment paths
- private configuration files
- private logs
- private calibration data
- non-public repository names unless explicitly approved
- customer, partner, or meeting-sensitive notes not intended for public release

## Public Writing Standard

Public documentation should be:

- technically credible
- honest about prototype status
- clear about safety boundaries
- specific enough to be useful
- broad enough to avoid exposing private implementation details

## Safe Language

Use:

- "Raspberry Pi 5-class edge hardware"
- "local edge runtime"
- "desktop operator app"
- "higher-reasoning cloud layer"
- "local motor authority"
- "safety-first runtime"

Avoid:

- private network locations
- operational credentials
- private service names that are not public products
- exact deployment commands
- private repo paths
- claims that imply field certification or finished production readiness
